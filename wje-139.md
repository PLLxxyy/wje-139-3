## 项目主要需求

### 项目概述

【全栈Web应用】供应链仓储出入库协同管理系统。面向供应链中转仓，支持多货主、多批次的精细化出入库管理，覆盖收货质检、上架存储、拣货复核、出库发货全链路，强调货主与仓库之间的协同操作。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿全栈）

1. **货主（Owner）** — 字段：货主名称、联系人、联系电话、邮箱、地址、结算方式（Monthly/PerOrder/Prepaid）、信用额度、当前欠款、状态（Active/Suspended）
   - 关联：1个货主 → 多个商品、多个入库单、多个出库单
   - 业务动作：添加货主、信用额度调整、暂停合作
   - **贯穿全栈**

2. **商品（Product）** — 字段：所属货主ID、商品名称、SKU、条形码、品类、规格、单位、保质期天数（可选）、存储要求（Normal/ColdChain/Dangerous/Fragile）、体积、重量
   - 关联：1个商品 → 多条库存记录
   - 业务动作：创建商品、更新信息
   - **贯穿全栈**

3. **入库单（InboundOrder）** — 字段：单号、所属货主ID、供应商名称、预计到货日期、实际到货日期、状态（Pending/Received/QCInProgress/Shelved/Completed）、质检员ID、库管员ID、备注
   - 关联：1个入库单 → 多个入库明细（InboundItem：商品ID、批次号、预期数量、实际数量、质检结果Pass/Fail、存放库位ID）
   - **贯穿全栈**

4. **出库单（OutboundOrder）** — 字段：单号、所属货主ID、收货方名称、收货地址、要求发货日期、实际发货日期、状态（Pending/Picking/Checking/Packing/Shipped/Completed）、拣货员ID、复核员ID、快递单号
   - 关联：1个出库单 → 多个出库明细（OutboundItem：商品ID、库位ID、应拣数量、实拣数量）
   - **贯穿全栈**

5. **库位（BinLocation）** — 字段：仓库区域（A/B/C/D）、货架号、层号、列号、容量（m³）、当前占用率、存储要求、状态（Available/Occupied/Reserved/Maintenance）
   - **贯穿全栈**

### 核心页面（≥ 4 个）

1. **仓库总览** `/dashboard` — 消费：InboundOrder + OutboundOrder + BinLocation + Owner
   - 今日收货/发货量、库位整体占用率（环形图）、各货主库存金额TOP10、待处理任务列表
   - 使用 `<StatusBadge>`, `<StatCard>`, `<OccupancyRing>` 共享组件

2. **入库管理** `/inbound` — 消费：InboundOrder + InboundItem + Product + BinLocation
   - 入库单列表（按状态Tab），创建入库单（选货主→添加商品明细），收货质检（逐项录入实收数量+质检结果），上架指派库位
   - 使用 `<StatusBadge>`, `<StepIndicator>`

3. **出库管理** `/outbound` — 消费：OutboundOrder + OutboundItem + Product
   - 出库单列表，创建出库单（选货主→选商品→系统推荐库位），拣货任务（逐项确认实拣数量），复核打包，发货确认
   - 使用 `<StatusBadge>`, `<StepIndicator>`

4. **库位管理** `/bin-locations` — 消费：BinLocation + Product
   - 库位网格视图（按区域分组，颜色表示占用率），库位详情（当前存放商品列表），批量创建库位
   - 使用 `<OccupancyRing>`, `<StatusBadge>`

5. **货主管理** `/owners` — 消费：Owner + InboundOrder + OutboundOrder + Product
   - 货主列表，货主详情（基本信息+库存统计+历史单据+账务概况）
   - 使用 `<StatusBadge>`, `<StatCard>`

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / WarehouseManager / QCInspector / Picker / Checker / Owner（货主角色，只能看自己的单据和库存）
   - 触达文件层：DB role表 + RBAC中间件 + 路由守卫 + 按钮/数据范围显隐 + 类型定义

2. **操作日志**
   - 记录收货、质检、上架、拣货、复核、发货每一步操作
   - 触达文件层：DB operation_log表 + 日志中间件 + 各service埋点 + 前端操作时间线 + 类型定义

3. **全局异常处理**
   - 触达文件层：后端异常过滤器 + controller异常 + 前端ErrorBoundary + API拦截器 + 错误码枚举

### 共享枚举/常量（≥ 2 组）

1. **入库单状态 InboundStatus**：Pending / Received / QCInProgress / Shelved / Completed
   - 后端：`backend/src/types/enums.ts`；前端：`frontend/src/types/enums.ts`

2. **出库单状态 OutboundStatus**：Pending / Picking / Checking / Packing / Shipped / Completed

3. **存储要求 StorageRequirement**：Normal / ColdChain / Dangerous / Fragile

4. **质检结果 QCResult**：Pass / Fail / Partial

### 共享前端组件清单（≥ 3 个）

1. **`<StatusBadge>`** — 所有页面
2. **`<StepIndicator>`** — 入库管理（收货→质检→上架）、出库管理（拣货→复核→发货）
3. **`<StatCard>`** — 仓库总览、货主管理
4. **`<OccupancyRing>`** — 仓库总览、库位管理
5. **`<EmptyState>`** — 所有列表空状态

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useInboundFlow`** — 封装入库全流程（收货→质检→上架的状态流转和数据更新）
2. **`usePagination`** — 通用分页
3. **`generateOrderNo`** — 单号生成工具（INB-YYYYMMDD-XXXX / OUT-YYYYMMDD-XXXX）

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色+数据范围校验（Owner只能看自己数据）
3. **`auditLogMiddleware`** — 操作日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`requestLoggerMiddleware`** — 请求日志

### 技术要求

- **英文短名（用于 Docker Compose）**：`supply-warehouse`

- **UI 库**：Element Plus

- **数据库**：PostgreSQL 15

### 部署要求

Docker Compose 一键部署。
- 顶层 `name: supply-warehouse`
- 不写 `version:` 字段
- PostgreSQL 命名卷 + healthcheck
- 后端 depends_on service_healthy
- 前端 `${FRONTEND_PORT:-80}:80`，后端 `${BACKEND_PORT:-3000}:3000`

### Git 要求

`git init` + `.gitignore`。
